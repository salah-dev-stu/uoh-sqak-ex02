## 1. Theoretical background

A Skill is a directory containing at minimum a `SKILL.md` with YAML frontmatter (`name`, `description`) and a markdown body. Anthropic's official guidance — surveyed in Prompt #9 — calls for:

- **Third-person, "pushy" descriptions** (avoid undertrigger)
- **1500–2000 word bodies** for substantive skills
- **Scope + Testing Expectations at the TOP** of the body
- **Progressive disclosure** — references loaded on demand via filesystem reads

For HW2 we make one deliberate divergence from Anthropic's standard: **ADR-002**. We do NOT use Claude's auto-discovery (where Claude reads frontmatter and decides which skill to invoke). Instead we **statically load** the SKILL.md body as a system prompt at child-process boot, via `--append-system-prompt`. Reason: each child is a single-purpose role (Pro / Con / Judge); auto-discovery introduces a risk of wrong-skill selection. The skill format is still used because (a) it's the lecturer-recognized idiomatic structure, (b) it's project-local per H17, (c) the frontmatter doubles as machine-readable metadata for tests.

The three skills correspond to the three agents:

- **`pro_skill`** — stance `AI=ORIGINALITY`, citations toward Klingemann / Edmond de Belamy / transformative use
- **`con_skill`** — stance `AI=REMIX_ONLY`, citations toward Stochastic Parrots / NYT v OpenAI / Chinese Room
- **`judge_skill`** — topic-blind moderator, references the **auto-generated** `debate_criteria.md` from the N7 web-search pre-flight (lec05 L1519-1528)

## 2. Functional requirements

| H-gate | Requirement |
|---|---|
| **H8** | `pro_skill` and `con_skill` produce genuinely different stances — Skills differ in body content, drift keywords, citation lists |
| **H17** | Skills live under `.claude/skills/` (project-local). NEVER `~/.claude/skills/`. Repo audit verifies. |
| **H19** | `judge_skill/SKILL.md` contains NO topic words (no "AI", "art", "original", "remix") |
| **N7** | `judge_skill/references/debate_criteria.md` is generated by `scripts/build_judge_criteria.py` web-searching for parliamentary debate criteria |

## 3. SKILL.md frontmatter contract

```yaml
---
name: pro-ai-originality-debater          # ≤64 chars, lowercase-kebab
description: Argues the affirmative position that AI agents can create
  genuinely original art, not merely remix human work. Use when this
  agent is spawned as the Pro debater. Cites Klingemann, Edmond de
  Belamy, transformative-use doctrine. Always references and rebuts
  the opponent's prior point. Never agrees with or concedes to Con.
                                          # ≤1024 chars, 3rd person, "pushy"
---
```

Validated at child-process boot: `load_skill_body()` parses the YAML frontmatter; if `name` or `description` is missing or malformed, raises `SkillFormatError`.

## 4. SKILL.md body required sections

```markdown
## Scope

(stance fixed; commitments; what this skill is not)

## Testing expectations

(every output is JSON; every output quotes opponent's text; every output
includes citations)

## Tactics

(specific arguments / counter-arguments to deploy; provenance for each)

## Drift signal keywords

(stance-violation phrases the Judge's DriftDetector will flag —
e.g., for Pro: "actually you're right", "I concede", "fair point",
"good argument", "I agree", "you've convinced me")

## Output format

(JSON shape required; example template)
```

Body target: 1500–2000 words. Sections 1 and 2 at the top per Anthropic best practice.

## 5. The N7 pre-flight (Judge criteria auto-build)

`scripts/build_judge_criteria.py` runs once before debate (skipped if cache hit). It:

1. Web-searches DDG for "parliamentary debate scoring criteria", "Lincoln-Douglas format", "Robert's Rules"
2. Synthesizes results into `.claude/skills/judge_skill/references/debate_criteria.md`
3. The Judge's `SKILL.md` body references this file: *"For your full scoring rubric, see references/debate_criteria.md."*

Lec05 L1519-1528 verbatim source: *"Tell Gemini: 'search the world for who's the #1 debate expert and what criteria he sets,' then take those criteria and give them as system prompt to the Father."*

## 7. Alternatives considered

| Alternative | Rejected because |
|---|---|
| Auto-discovery via Claude's standard SKILL.md frontmatter mechanism | ADR-002 — risk of wrong-skill selection in a single-purpose child |
| Global skills under `~/.claude/skills/` | H17 explicit — *"don't put it in global skills"* |
| Hardcoded Judge criteria | N7 lecturer-specific request; sourced criteria earn originality bonus |
| Skills in `src/agent_debate/skills/` (Python package) | Skills are markdown documents, not code; idiomatic location is `.claude/skills/` |
| Skill bodies in English only (skip Hebrew) | Decision: English-only (matches H12 + lecturer's reading preference + token efficiency per lec04 §3.2) |
